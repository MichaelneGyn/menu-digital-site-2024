═══════════════════════════════════════════════════════════
  📋 PASSO A PASSO
═══════════════════════════════════════════════════════════

🔧 FAZER MUDANÇAS (SEMPRE NO DEVELOP):

  1. git checkout develop
  2. git pull origin develop
  3. (faz alterações nos arquivos)
  4. npm run pre-deploy
  5. git add .
  6. git commit -m "descreva o que fez"
  7. git push origin develop
  8. Aguarda 2-3 minutos

─────────────────────────────────────────────────────────

🧪 TESTAR NO STAGING:

  URL: https://...git-develop...vercel.app
  
  ✅ Login funciona?
  ✅ Dashboard abre?
  ✅ Cardápio funciona?
  ✅ Sua mudança funcionou?
  ✅ Sem erros no console? (aperte F12)
  
  SE TUDO OK ✅ → Vai pro próximo passo
  SE ALGO FALHOU ❌ → Volta e corrige

─────────────────────────────────────────────────────────

🚀 LIBERAR PARA CLIENTES (SÓ DEPOIS DE TESTAR!):

  1. git checkout master
  2. git pull origin master
  3. git merge develop
  4. git push origin master
  5. Aguarda 2-3 minutos
  6. Testa DE NOVO em: https://...git-main...vercel.app
  7. Se OK → Clientes podem usar! 🎉

═══════════════════════════════════════════════════════════
  🚨 EMERGÊNCIA - ROLLBACK
═══════════════════════════════════════════════════════════

Se algo quebrou em produção:

  1. Acesse: https://vercel.com/dashboard
  2. Deployments
  3. Encontra deploy anterior (que funcionava)
  4. ... → Promote to Production
  5. PRONTO! Voltou!
